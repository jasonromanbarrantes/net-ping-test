LLS – 15-Min Connectivity Ping

Client-to-Server network path verification

Purpose

This diagnostic verifies end-to-end connectivity and latency from the interpreter’s workstation and local network to the LanguageLine service infrastructure at the selected location. Results help LLS IT confirm whether performance issues originate on the client network path or beyond.

One-time setup required: Click Register LLS Diag below to enable the llsdiag:// protocol on this PC. After that, starting a test is a single click.

Buttons launch the local test via the registered llsdiag:// protocol.

What happens on click

  1. Your browser calls llsdiag://run?ip=<IP>&dur=900.
  2. Windows launches PowerShell with a secure, predefined command that fetches and runs Runner.ps1 from this site.
  3. The script pings the selected IP ~1/second for 15 minutes, saves raw logs, summary, and a ZIP in %TEMP%, and opens the folder at the end.

Privacy: All packets originate from the interpreter’s PC (their network). No credentials are used.
